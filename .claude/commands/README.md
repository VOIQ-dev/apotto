# カスタムスラッシュコマンド

このディレクトリには、プロジェクト専用のカスタムスラッシュコマンドが含まれています。

## 利用可能なコマンド

### 📚 ドキュメント更新コマンド

#### `/update-api-spec` - API仕様書を更新

営業メール生成APIのAPI仕様書（`docs/api-sales-copy-specification.md`）を最新のコード実装に基づいて自動更新します。

**使い方**:

```
/update-api-spec
```

**更新内容**:

- エンドポイント詳細
- リクエスト/レスポンス仕様
- パラメータ一覧
- 処理フロー
- エラーハンドリング

---

#### `/update-api-usage` - 操作マニュアルを更新

営業メール生成APIの操作方法マニュアル（`docs/api-sales-copy-usage.md`）を最新のAPI仕様と実装に基づいて自動更新します。

**使い方**:

```
/update-api-usage
```

**更新内容**:

- 基本的な使い方
- 実装例（JavaScript, TypeScript, Python）
- ストリーミング版の使い方
- よくあるユースケース
- トラブルシューティング
- ベストプラクティス

---

#### `/update-api-docs` - すべてのAPIドキュメントを一括更新

API仕様書と操作マニュアルの両方を一括で自動更新します。

**使い方**:

```
/update-api-docs
```

**更新内容**:

- API仕様書の全セクション
- 操作マニュアルの全セクション
- 変更内容のサマリーレポート

---

## 使用例

### 単一ドキュメントの更新

API仕様書のみを更新したい場合：

```
/update-api-spec
```

操作マニュアルのみを更新したい場合：

```
/update-api-usage
```

### 一括更新

API実装を変更した後、関連するすべてのドキュメントを更新したい場合：

```
/update-api-docs
```

## コマンドの仕組み

スラッシュコマンドは、`.claude/commands/` ディレクトリ内のマークダウンファイルとして定義されています。

- ファイル名（拡張子なし）がコマンド名になります
- ファイル内容がClaude Codeへの指示（プロンプト）になります

例：

- `update-api-spec.md` → `/update-api-spec` コマンド
- `update-api-usage.md` → `/update-api-usage` コマンド

---

## 新しいコマンドの追加方法

1. `.claude/commands/` ディレクトリに新しいマークダウンファイルを作成
2. ファイル名をコマンド名にする（例: `my-command.md`）
3. ファイル内にClaude Codeへの指示を記述
4. コマンドが即座に利用可能になります

**例**:

`.claude/commands/review-code.md` を作成：

```markdown
現在のブランチのコード変更をレビューして、以下の観点でフィードバックしてください：

1. セキュリティ上の問題
2. パフォーマンスの改善点
3. コーディング規約の遵守
4. テストの不足
```

これで `/review-code` コマンドが使えるようになります。

---

## ベストプラクティス

1. **明確な指示を記述**: コマンドの目的と実行内容を具体的に記述する
2. **手順を明示**: 複雑なタスクは手順を番号付きリストで記述する
3. **成果物を明確に**: 何を生成・更新するかを明記する
4. **コンテキストを提供**: 必要なファイルパスや参照先を指定する

---

## トラブルシューティング

### コマンドが認識されない

- ファイル名に空白が含まれていないか確認
- ファイルが `.claude/commands/` ディレクトリに配置されているか確認
- ファイル拡張子が `.md` であることを確認

### コマンド実行後に期待した結果が得られない

- コマンドファイルの指示内容を見直す
- 具体的なファイルパスや手順を追加する
- 実行するタスクを小さく分割する
